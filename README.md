# 📸 Photo_take：零件連續拍攝與自動儲存系統

本專案透過 **Google Colab** 結合網頁前端 **JavaScript** 技術，實現利用本機視訊鏡頭（或外接虛擬相機 App）進行定時、定量的連續工業零件照片拍攝。所有拍攝的照片將會自動同步儲存至使用者的 **Google 雲端硬碟 (Google Drive)** 中，非常適合用於工業零件 AI 數據集收集、標註與品質檢測影像存檔。

---

## 🛠 手機鏡頭連接指南 (使用 iVCam)

為了獲得更高畫質的零件照片，建議使用手機的**後置鏡頭**代替筆電內建的低畫質鏡頭。本專案推薦並完整支援使用 **iVCam** 進行連線。

### 步驟 1：下載與安裝
* **電腦端 (Windows)**：請至 [e2eSoft iVCam 官方網站](https://www.e2esoft.com/ivcam/) 下載並安裝 Windows 用戶端軟體。
* **手機端 (Android / iOS)**：請在 App Store 或 Google Play 商店搜尋並下載 `iVCam` App。

### 步驟 2：裝置連線
1. 將你的電腦與手機連線至**同一個 Wi-Fi 網域**（或使用 **USB 傳輸線**連接電腦與手機，傳輸更穩定且延遲低）。
2. 先開啟電腦端的 iVCam 軟體。
3. 開啟手機端的 iVCam App，軟體會自動搜尋並完成連線。連線成功後，電腦畫面上會立刻出現手機相機的即時預覽畫面。
4. **(建議)** 在手機端設定中，將**拍攝焦距固定（Lock Focus）**、並調整至最高的解析度（如 1080p 或 4K），以利零件細節留存。

### 步驟 3：三種方法確認連線成功

#### 1. 檢查 iVCam 電腦端軟體畫面（最直覺）
* **未連線成功**：開啟電腦上的 iVCam 軟體，若畫面顯示 `Waiting for iVCam connection...`（等待 iVCam 連線），代表尚未連上手機。
* **連線成功**：電腦端的 iVCam 視窗會立刻顯示手機相機拍到的即時畫面。此時手機畫面的右下角或上方也會顯示連線成功圖示。

#### 2. 檢查 Windows「裝置管理員」（確認系統抓到驅動）
有時候軟體卡住，你可以確認 Windows 系統底層有沒有把手機識別為視訊鏡頭：
* 在 Windows 開始選單（或工作列放大鏡）搜尋 **「裝置管理員」** 並開啟。
* 尋找 **「影像裝置」**（Cameras 或 Imaging devices）這一欄並展開。
* 如果有看到 **`e2eSoft iVCam`**，代表 Windows 系統已經成功將它識別為可用的虛擬視訊鏡頭。

#### 3. 使用 Windows 內建的「相機」App 測試
這是最保險的驗證方式，能確保其他軟體（如瀏覽器、Colab）也能正常讀取：
* 在 Windows 搜尋列輸入 **「相機」**，打開 Windows 內建的相機應用程式。
* 點擊相機 App 右上角或側邊的 **「切換相機」按鈕**（通常是一個相機帶有旋轉箭頭的圖示）。
* 切換後，如果畫面成功轉成手機鏡頭拍到的畫面，那就百分之百確定連線成功！

## 🚀 使用步驟與操作說明

請在 Google Colab 環境中依序執行以下儲存格：

### 1. 基礎初始化與路徑建立
執行第一個儲存格，系統會自動在 Google Drive 中建立預設的相片儲存資料夾。

### 2. 雲端硬碟掛載與環境設定
執行主程式時，Colab 與瀏覽器會跳出權限請求視窗，請務必進行以下操作：
* **Google Drive 授權**：請允許掛載 Google Drive 權限，以便儲存照片。
* <img width="681" height="320" alt="圖片" src="https://github.com/user-attachments/assets/4774b37b-8552-460f-947a-68737c1b633e" />

* **相機權限**：瀏覽器彈出「Colab 想要使用您的照相機」時，**請點選「允許」**。*（請確保此時 Windows 上的 iVCam 已成功連線手機）*

### 3. 模式選擇與拍攝
本程式提供兩種連續拍攝模式，請根據需求選擇對應的儲存格執行：

#### 🔹 模式 A：定時定量拍攝 (指定總時間)
* **程式位置**：第二個程式儲存格。
* **自訂參數**：
  * `DURATION`: 總拍攝秒數 (預設 `23` 秒)。
  * `INTERVAL`: 每次拍照間隔秒數 (預設 `2.0` 秒/張)。
* **操作**：點擊執行後，網頁左上方會出現即時預覽視窗，並倒數計時自動拍照。亦可隨時點擊 **`🛑 停止拍攝`** 提前結束。

#### 🔹 模式 B：特定鏡頭鎖定拍攝 (推薦 iVCam 使用)
* **程式位置**：第四個程式儲存格 (核心優化功能)。
* **特點**：此模式會自動搜尋名稱包含 `iVCam` 的外接相機裝置，**避免 Colab 誤用筆電內建的前置鏡頭**。
* **自訂參數**：
  * `TARGET_CAMERA_NAME`: 設為 `"iVCam"`。
  * `MAX_IMAGES`: 預期拍攝的總張數 (預設 `5` 張)。
  * `INTERVAL`: 每次拍照間隔秒數 (預設 `2.0` 秒)。
* **⚠️ 第一次執行注意**：模式 B 在啟動時會先短暫喚醒預設鏡頭，以向瀏覽器取得完整的鏡頭清單（解決裝置名稱可能為空白的問題），接著會自動切換至 iVCam 鏡頭。請在畫面上確認手機畫面對準零件後，再開始移動或更換零件。

## 🏗 技術架構與運作原理 (Code Explanation)

由於 Google Colab 的 Python 程式碼實際上運行於遠端的 Google 雲端伺服器，**無法直接存取使用者本機端的硬體裝置**。因此，本專案採用了 Python 與前端 JavaScript 混合的異步通訊架構來繞過此限制：

[Google Colab 後端 (Python)]
│
├─ 1. 發送指令與自訂參數 (總時間、間隔)
▼
[使用者瀏覽器前端 (JavaScript)] ── 2. 請求本機相機權限 (iVCam/鏡頭)
│
├─ 3. 即時渲染視訊畫面 ()
├─ 4. 定時截圖轉為畫質 0.98 的 Base64 數據 ()
│
▼ (透過 google.colab.kernel.invokeFunction 通道)
[Google Colab 後端 (Python)]
│
├─ 5. 接收 Base64 圖片數據並解碼
└─ 6. 自動存入 Google 雲端硬碟 (Google Drive)

* **參數初始化**：Python 端定義儲存資料夾、拍攝總時長、拍照間隔與畫質。
* **網頁端喚醒鏡頭**：透過 `IPython.display.Javascript` 將前端程式碼注入瀏覽器，使用網頁標準 API (`navigator.mediaDevices.getUserMedia`) 呼叫視訊來源，並優先請求 $1920 \times 1080$ (1080p) 高清畫質與固定 30fps 幀率。
* **即時畫面截取**：網頁端建立一個固定在畫面左上角的浮動 HTML5 `<video>` 預覽視窗。當 `setInterval` 定時器啟動時，會定時將視訊當前幀繪製到隱藏的 `<canvas>` 畫布上，並導出為經過高畫質壓縮的 JPEG Base64 字串。
* **跨環境雙向通訊**：利用 Colab 內建的 JS 擴充套件通道，呼叫 Python 端的 `save_image` 函數，將圖片資料即時回傳給後台。
* **後端異步存檔**：Python 端收到資料後，使用 `PIL (Pillow)` 影像處理庫將其還原為二進位圖檔，並以時間戳記命名即時寫入雲端硬碟。

## ⚙️ 核心自訂參數說明

您可以直接在程式碼頂部的 **`# 設定區`** 修改以下變數，以適應不同的零件採集情境：

| 變數名稱 | 預設值 | 說明 | 適用情境建議 |
| :--- | :--- | :--- | :--- |
| `SAVE_FOLDER` | `'/content/drive/MyDrive/Camera_Continuous/'` | 照片在 Google 雲端硬碟的儲存路徑 | 可改為不同的零件產品名稱作為資料夾分類 |
| `DURATION` | `23` | 總共持續拍攝的秒數 (模式 A) | 若零件數量極多，可放大至 `300` 秒以上持續作業 |
| `INTERVAL` | `2.0` | 每次自動截圖拍照的間隔時間（秒） | 更換零件熟練後，可縮短至 `1.0` 或 `0.5` 秒提高採集效率 |
| `QUALITY` | `0.99` | 輸出的 JPEG 圖片品質比例 ($0.0 \sim 1.0$) | 建議保持 `0.99`，以完整保留零件表面瑕疵與型號細節 |

## 📂 照片儲存路徑與命名規範

所有拍攝的照片將會以時間戳記與流水號自動命名：
* **命名格式**：`snapshot_時分秒_張數.jpg` (例如：`snapshot_173045_1.jpg`)
* **儲存路碟路徑**：`我的雲端硬碟 / Camera_Continuous /`
